A. Memory and Crow

题目大意

有一个数列\(b[1],b[2],…,b[n]\)，另一个数列\(a[i]=b[i]-b[i+1]+b[i+2]-b[i+1]+…\)，现已知数列\(a\)，求数列\(b\)？

思路 - 模拟

很容易就能发现：\(b[i]=a[i]+a[i+1]\)，\(b[n]=a[n]\)。

B. Memory and Trident

题目大意

有一个只含有\(LRUD\)四个字符的字符串，每次可以将其中一个字符变为任意一个字符，求最少变换多少次，能使\(L\)出现的次数等于\(R\)出现的次数，\(U\)出现的次数等于\(D\)出现的次数，无法满足题意输出\(-1\)？

思路 - 贪心

首先可以想到输出\(-1\)一定是字符串长度为奇数时。
当字符串长度为偶数时，可以贪心地将\(L\)和\(R\)中的多的变成少的，\(U\)和\(D\)同理，设两种的差的绝对值分别为\(x\)和\(y\)，则答案为\(\frac {x+y} {2}\)。

C. Memory and De-Evolution

题目大意

问把一个边长为\(x\)的等边三角形变成变长为\(y\)的等边三角形最少要经过多少次操作？
进行的操作如下：每次可以将一条边变成任意长度，但必须保证操作后的三角形面积为正。

思路 - 贪心

题目规定\(y < x\)，可以考虑\(y\)变成\(x\)，这样更简单。
每次贪心地选择三角形中最短的一条边\(a\)，将其变成\(min(x,b+c-1\)，直至其变成变长为\(x\)的等边三角形。

D. Memory and Scores

题目大意

\(Memory\)初始有\(a\)分，\(Lexa\)初始有\(b\)分，他们轮流随机选择\(t\)个数，每个数的范围是\([-k,k]\)，求最终\(Memory\)的分数大于\(Lexa\)的分数的不同方案数？方案数不同当且仅当至少有一次其中至少一个人选了不同的数。

思路Ⅰ - DP && sliding window technique

很容易就能想到\(dp\)，\(dp[i][j]\)表示第\(i\)次选数后，\(Memory\)的分数减去\(Lexa\)的分数为\(j\)时的方案数，第二维大小为\(2kt\)，第一维大小为\(t\)，所以状态转移必须是\(O(1)\)的。由于一定是上一次一段连续的区间会更新到当前的\(dp\)值，所以可以维护一个前缀和。
以上是没注意到每个\(dp\)值的权重不一样时想到的，结果一直跑不对，仔细一想还有权重，然后就想不到该如何做了。。。
看了题解才发现可以使用\(sliding window technique\)，提前处理好第一个区间的带权和，然后每次可以\(O(1)\)维护区间带权和。

E. Garlands

题目大意

在\(n*m\)的格子中，有\(k\)条链，第\(i\)条链的有\(len[i]\)个灯泡，每个灯泡在一个确定的格子中，链上有一个开关，没按下一次，所有灯泡状态均变化，初始所有灯泡都亮着（亮时权值为\(w[i][j]\)，暗时权值为\(0\)），有\(q\)次操作，每次可以按下第\(i\)条链的开关，或者询问一个区域的权值和。

思路 - 树状数组

看了题解才注意到询问次数不超过\(2000\)次，所以可以先预处理出每一条链亮着时对每个询问的贡献，可以用树状数组维护，然后再按照操作次序更改每条链的状态，询问时判断链是否对当前询问有贡献即可。
